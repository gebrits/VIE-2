A Mapping provides functionality to map context knowledge to Javascript objects. This can either be accomplished by using the default functionality of the filter method, or by overwriting this.
In general, the mapping function should never be called directly but only through the core. 

             		           
                               
                            
                 ¶               
               
Constructor(id, [options]): The constructor needs an id of type string.
Exceptions are thrown if either no 'id' is given or the id is not of type string.
Options are optional and may be passed after the id to the constructor.
             		                            
Mapping = function(id, options) {
	if (id === undefined) {
		throw "The mapping constructor needs an 'id'!";
	}
	
	if (typeof id !== 'string') {
		throw "The mapping constructor needs an 'id' of type 'string'!";
	}
	this.id = id;
	
	this.options = options;

In the default functionality of this method, we can simply pass
the option.mapping object to the constructor that is automatically parsed and
used by the filter function. Let's have a look
at the place mapper:
    
new Mapping('place', {
        mapping :  {
            'type' : {
                'rdfa' : {
                    'type' : 'rdf:type', 
                    'value' : 'dbonto:PopulatedPlace'
                }
            },
            'name' : {
               'rdfa' : ['rdfs:label', 'foaf:name']
            },
            'long' : {
                'rdfa' : ''
            },
            'lat' : {
                'rdfa' : 'geo:lat'
            }
       }
    });

We can see that the mapping option is a key value hashmap which needs
a pre-given syntax. It is connector-specific in it's mapping and in the example,
'rdfa' is the id of the RDFa connector.
Filtering for entity types is specified in the 'type' key which needs
for each connector a 'type' and 'value' key, where
'value' can also be an array of strings. In the example, we only search
for elements that are of type PopulatedPlace from the dbPedia ontology.
Notice: As long as VIE^2 was initialized with proper namespace mappings, these mappings
can be used through the whole framework consistently.
In the example, the context is scanned for all places and for each found
entity, a new Javascript object is allocated with the keys name, long, lat.
As we can see, the mapping also either applies to strings or arrays of strings. 
Full URIs need to be enclosed by the < and > symbols.
